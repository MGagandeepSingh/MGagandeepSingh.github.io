Signed vs Unsigned Data Types

Updated: 31-Dec-2022
Author: Gagandeep Singh

Programming has two types of data types

  1. Signed
  2. Unsigned

The term unsigned in computer programming indicate a variable that can hold only positive values.
Whereas signed data type indicates that both negative and positive values can be represented.

As you already know numbers are stored in binary format internally. We will understand how signed and unsigned types work internally using the byte data type in Java. You can get more insight into Java's primitive data types and understand how much memory they take and what values can they hold.

Signed Data Types

Byte data type in Java consists of 1 byte which is equal to 8 bits. Each bit is capable of holding two values 0 & 1.

The above binary value represent the number 127, we have got this number by adding all the bit values that are high (1).

64 + 32 + 16 + 8 + 4 + 2 + 1 = 127

And if the last bit was also high, we would have got the value as 128 + 127 = 256

This is unsigned bit for you, where in all the bits are used to represent the positive numbers. Since all bits can be used to represent a number so the range it can provide is from 0 - 255.

Unsigned Data Types

What if we want to make use of the negative numbers?

For using negative numbers what we can do is we can use on the bit to represent negative number (which is the last bit, 8th bit, index 7 or most significant bit)

Now this means that when the most significant bit is high (1), the number would be a negative number. It is safe to say it will be positive otherwise. Another fact is that now we can only use first 7 bits (0 - 6) to represent the positive numbers and thus the maximum value that we can now get is 127, Hence the range of byte data type is -128 to 127.

This is what is called as signed bit and Java only uses signed data types except for char data type.

Why Java doesn't support unsigned data types

Java only supports signed data types as unsigned data types do not belong in Java programming language because they add complexity to the type system, language and VM specifications.

Java creators really wanted to develop a language which should be simple and they wanted to ensure security remains top priority. Both these principles led to the decision of not including unsigned data types in Java.

Everybody thought that unsigned types make everything more complicated and nobody really needs them as we can achieve the functionality with small amount of knowledge and effort.
